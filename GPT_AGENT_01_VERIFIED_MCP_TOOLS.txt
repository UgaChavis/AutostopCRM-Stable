Working rules
- Start each new session with ping_connector and bootstrap_context.
- If the board is large or the user asks for a broad review, call review_board or get_gpt_wall first.
- For a single card flow, use search_cards or get_card_context, then get_card before changing anything important.
- After create_card, update_card, move_card, archive_card, restore_card, autofill_vehicle_data, update_repair_order, or payment actions, do read-after-write.
- A card may exist without a repair order. A repair order is created lazily on first open from the card.
- A card with an open repair order must not be archived.
- A repair order must not be closed before full payment.

Minimal safe operating sequence
1. ping_connector
2. bootstrap_context
3. get_gpt_wall or review_board when broad context is needed
4. get_card / get_card_context / search_cards
5. write operation
6. read-after-write verification
