AUTOSTOP CRM REPAIR ORDER AND PAYMENTS PLAYBOOK

Purpose
- This file describes how the agent should reason about cards, repair orders, payments, and archive safety.

Lifecycle model
- A card may exist without a repair order.
- A repair order must not exist separately from a card.
- A repair order is created lazily on first open from its card.
- Reopening the repair-order module for the same card must return the existing order.

Safe repair-order flow
1. Read the card.
2. Open the repair order with get_repair_order when the task truly needs it.
3. If it is the first open, allow lazy creation.
4. Read the returned order before editing works, materials, notes, or totals.
5. After any repair-order write, read it again.

Repair-order command map
- get_repair_order: open or lazily create the order for the card
- update_repair_order: edit common fields
- replace_repair_order_works: replace works list deterministically
- replace_repair_order_materials: replace materials list deterministically
- autofill_repair_order: enrich the order from the card context
- get_repair_order_text: retrieve printable text view
- set_repair_order_status: open or close the order

Payment rules
- An unpaid or partially paid repair order must not be closed.
- Before closing, confirm the order is fully paid.
- Payment operations should be reflected through cashboxes and transactions.
- After payment write, read the order or cashbox state again.

Archive rules tied to repair orders
- Card with no repair order: archive allowed.
- Card with closed repair order: archive allowed.
- Card with open repair order: archive forbidden.

Inconsistent legacy state handling
- If an archived card with an open repair order is found, treat it as inconsistent legacy data.
- Do not silently continue normal operations as if the state were valid.
- Prefer a controlled error or a guided close-first workflow.

Minimal payment-close sequence
1. get_repair_order
2. verify payment state
3. create_cash_transaction if payment is needed
4. get_repair_order or get_cashbox
5. set_repair_order_status to closed
6. get_repair_order again
